# Проектирование взаимодействия

## Внешнее взаимодействие

### Интеграция с DSP

Интеграция с DSP предполагает применение OpenRTB протокола так как OpenRTB является промышленным стандартом RTB-индустрии. DSP ожидают именно этот формат.

Транспорт: HTTPS (зашифрованный канал)
Формат: JSON (стандарт OpenRTB)
Шлюз: DSP Gateway (выполняет задачи терминации TLS, rate limiting, маршрутизации)
Роут: `/bid_request`

## Внутреннее взаимодействие

Критичный путь не содержит синхронных запросов, логика аукциона изолирована в одной системе - RTB Service

Все взаимодействие предполагает асинхронную коммуникацию на основе событий с использованием Kafka.

- Обработка событий ставки кампаний из Campaign Service
- Обработка событий изменения правил таргетинга из Campaign Service
- Обработка событий изменения кампаний из Campaign Service

Идемпотентность по campaign_id: гарантия доставки At-least-once
