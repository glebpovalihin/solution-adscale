# RTB Service: границы, API, зависимости и модель данных

## 1. Введение

RTB Service — первый сервис, выделяемый из монолита на Этапе 1 (0–3 мес.) согласно [ADR-001](../Task1/adr/ADR-001-monolith_to_services.md).

Обоснование:
| Фактор | Значение |
|---|---|
| Приоритет | **Наивысший** — сервис находится на critical path |
| Latency-чувствительность | Максимальная — определяет P95 всего аукциона |
| Текущая проблема | Синхронные вызовы к единой PostgreSQL в hot path |
| Эффект от выделения | Изоляция БД, независимое масштабирование, Redis-кэш, снижение P95 |
**Цель:** выдержать 18 000 RPS с P95 не более 100 ms (промежуточный этап) и не более 80 ms (в целевом решении).

## 2. Границы сервиса (Bounded Context)

Включает:

- Приём и обработка OpenRTB bid request
- Идентификация пользователя
- Подбор кандидатов по таргетингу
- Проведение аукциона, определение победителя

## 3. API

#### Внешнее апи:
- /bid_request [REST] - Приём bid request, возврат bid response для DSP-партнера

Почему REST - внешняя интеграция с использованием OpenRTB протокола.

#### Внутреннее апи:
- Обработка событий ставки кампаний из Campaign Service
- Обработка событий изменения правил таргетинга из Campaign Service
- Обработка событий изменения кампаний из Campaign Service

Почему Kafka события - минимизиация влияния на доступность системы аукциона сторонними системами.

## 5. Модель данных

Система предполагает наличие трех компонентов:

- PostgreSQL (Master) - для записи информации по кампаниям, ставкам, таргетингу, результатам аукциона.
- PostgreSQL (Replica) - для чтения при cache miss
- Redis - для оптимизации операций чтения

#### Модель данных PostgreSQL:

|Таблица|Связи|Описание|
|---|---|---|
|campaings	| has_many: bids, targeting_rules | Кампании (id, название, id рекламодателя, статус) |
|bids	| has_one: campaings | Ставки (id, id кампании, значение) |
|targeting_rules	| has_one: campaings | Правила таргетинга (id, id кампании, тип правила, значение правила) |

#### Модель данных Redis:

|Ключ|Тип|TTL|Описание|
|---|---|---|---|
|active_campaigns	| Set	| 5 мин	| Список активных campaign_id, кампании имеют ends_at при выход за который должны переставать быть активными (для candidate matching)|
|bid:{campaign_id}|	Hash |	1 час	| Ставка кампании, меняется редко, обновляется из Kafka |
|targeting:{campaign_id} |	Set + JSON	| 1 час	| Правила таргетинга, меняется редко, обновляется из Kafka |
