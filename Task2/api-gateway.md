# API Gateway для DSP-интеграции

В качестве Api Gateway (DSP Gateway) был выбран Envoy Proxy так как обладает гибкой конфигурацией, решает задачи: observability, Circuit Breaking, mTLS и интеграции с Istio.

## Задачи DSP Gateway

### Маршрутизация запросов

На основе встроенного механизма API Gateway

### Rate Limiting

Ограничение частоты запросов для защиты RTB Service от перегрузки и соблюдения SLA с DSP.

Rate Limiting по API Token DSP-партнера, лимит равен максимальной нагрузке * 1.5 (18 000 rps * 1.5 = 27 000 rps)

### Аутентификация

На основе API Token DSP партнера, дополнительно можно применить  mTLS для предотвращения возможности доступпа при утечке токенов.

### Circuit Breaker

Circuit Breaker предотвращает каскадные сбои - если RTB Service деградировал, DSP Gateway своевременно отключает его от трафика.

Circuit Breaker предлагается встроить в API Gateway, так как входящие запросы удовлетворяют требованию применимости только к HTTP запросам (Envoy Proxy позволяет настроить данное решение).

В качестве стратегии Retry запросов предлагается "Экспоненциальный backoff с джиттером" чтобы избежать одновременных повторов и распределить повторы во времени.

### Observability

Envoy предоставляет телеметрию as is на основе которой можно построить алертинг и сбор метрик (rps, P95/P99 latency, error rate).
