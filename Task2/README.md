# Проектирование RTB и интеграции с DSP

1. Проектирование сервиса ставок
Выделите сервис ставок из монолита, опишите его границы и API, определите зависимости от других сервисов и спроектируйте его модель данных.

2. Проектирование взаимодействия
Определите протоколы и паттерны взаимодействия.

3. API Gateway для DSP-интеграции
Спроектируйте API-слой для новой DSP-площадки: маршрутизацию запросов на ставки, ограничение частоты запросов, аутентификацию, паттерн автоматического размыкателя и мониторинг времени отклика.

4. Паттерны надёжности
Опишите, как обеспечить надёжность: паттерн Circuit Breaker для вызовов зависимых сервисов, повторные попытки с экспоненциальной задержкой, идемпотентность финансовых операций и резервные стратегии (кэшированные или дефолтные ставки).
