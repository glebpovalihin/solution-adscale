# Потоковая обработка данных

### Правила обработки

Системно применяем интеграцию через Kafka события.
Перечень допустимых типов событий:
- Фактические данные - информация о произошедшем. Это неизменное событие в определенный момент времени
- Change data capture (CDC) - доставляет изменения по какой либо сущности.
- Команды - показывает что должно быть исполнено в системе

На текущий момент описания решения все топики передают только cdc данные.

Для описания схемы событий используется proto:
- строгая типизация
- компактный размер
- автогенерация кода

### Топики

|Топик|Овнер|Потребители|Retention|Гарантия доставки|
|--|--|--|--|--|
|Ставки|Campaign Service|RTB Service|3 дня + компакт событий по ключу|At-least-once (идемпотентность по campaign_id)|
|Таргетинг|Campaign Service|RTB Service|3 дня + компакт событий по ключу| At-least-once (идемпотентность по campaign_id)|
|Кампании|Campaign Service|RTB Service, Delivery Service|3 дня + компакт событий по ключу| At-least-once (идемпотентность по id)|
|Показы|Statistic Service|Analytics Service, Financial Service|7 дней| At-least-once (идемпотентность по event_id)|
|Клики|Statistic Service|Analytics Service, Financial Service|7 дней| At-least-once (идемпотентность по event_id)|
