# Данные, масштабирование и отказоустойчивость

1. Database per Service
Определите стратегию данных для каждого сервиса

2. Масштабирование баз данных
Опишите стратегию масштабирования, чтобы понять, для каких сервисов применить репликацию Master-Slave, какие данные и по какому ключу шардировать, нужны ли read replicas для сервиса аналитики и где применимо разделение ответственности команд и запросов (CQRS).

3. Кэширование
Спроектируйте стратегию кэширования. В частности, что кэшировать (кампании, бюджеты, ставки), где разместить Redis, каковы TTL и стратегия инвалидации, а также как организовать прогрев кэша (cache warming).

4. Потоковая обработка данных
Опишите архитектуру потоковой обработки событий: топики Kafka, схему событий (Avro или JSON), группы потребителей и политику хранения.

5. Отказоустойчивость данных
Опишите отказоустойчивость данных: RPO и RTO для каждого сервиса, стратегию резервного копирования, и соответствие принципам 12-факторных приложений.
